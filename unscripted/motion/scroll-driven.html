Scroll-driven animations

Baseline: Limited availability · Chrome 115+, Firefox (flagged)

Replaces: scroll listeners for progress bars and parallax, and IntersectionObserver for reveal-on-scroll. The gradient bar at the top of this page and the sections below are driven entirely by animation-timeline.

Scroll down. The progress bar tracks scroll(); each section animates in on its own view() timeline.

view() — the IntersectionObserver killer

This section's entry animation runs as it crosses the viewport — mapped to scroll position, not to time. Scroll back and forth: the animation scrubs in both directions, which an IntersectionObserver could never do.

scroll() — the progress bar

The bar at the top maps document scroll to a scale keyframe. One animation, one timeline, zero event handlers — and it runs off the main thread.

animation-range

animation-range: entry 0% entry 60% confines the effect to the moment the element enters — the declarative version of every threshold configuration ever passed to an observer.

The entire implementation

.progress {
  animation: grow linear both;
  animation-timeline: scroll();
}

.reveal {
  animation: rise linear both;
  animation-timeline: view();
  animation-range: entry 0% entry 60%;
}
