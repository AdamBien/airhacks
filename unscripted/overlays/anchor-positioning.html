Anchor positioning

Baseline: Limited availability · Chrome 125+

Replaces: Floating UI, Popper, and every hand-written getBoundingClientRect() calculation — including the flip-when-near-the-edge logic, which position-try-fallbacks handles declaratively.

Positioned with position-anchor + position-area — no positioning library.

On narrow viewports I flip to the other side via position-try-fallbacks.

The entire implementation

button { anchor-name: --a; }

.tip {
  position: fixed;
  position-anchor: --a;
  position-area: block-start;
  position-try-fallbacks: flip-block;
}

The tooltip tracks its anchor through scrolling and resizing, and repositions itself when it would overflow the viewport — behavior that previously required a resize observer, a scroll listener, and a positioning engine.
