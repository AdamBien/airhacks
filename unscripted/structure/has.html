:has()

Baseline: Widely available · since 2026-06

Replaces: state classes toggled by change listeners — the container.classList.toggle('selected') pattern. The parent styles itself by asking about its children.

Nothing selected yet — the hint you are reading is hidden by :has(input:checked) the moment you pick a plan.

The entire implementation

.plan:has(:checked) { border-color: var(--accent); }

.picker:has(input:checked) .hint { display: none; }

Selection state lives where it always did — in the form controls. CSS now reads it upward, so no JavaScript has to copy it into class names.
