Customizable <select>

Baseline: Limited availability · Chrome 135+

Replaces: custom dropdown components — and with them the keyboard navigation, ARIA wiring, and mobile behavior those components always get subtly wrong. The element stays a native <select>.

In browsers without support this degrades to the fully functional native select.

The entire implementation

select,
select::picker(select) {
  appearance: base-select;
}

select::picker(select) { /* style the listbox */ }
option:checked { /* style the selection */ }

Opting in with appearance: base-select makes the trigger and the picker stylable and allows real markup inside options — the swatches here are <span> elements.
